Semantic HTML

Semantic HTML helps screen readers and other assistive technologies understand the content and navigate it more effectively
By using these semantic HTML elements appropriately, you provide meaning and structure to your webpage, which benefits both users and assistive technologies like screen readers. This makes it easier for users to understand and navigate your content, improving accessibility and usability.

Heading

            
            <!DOCTYPE html>
            <html>
            <body>
            <h1>Heading 1, the title/primary heading</h1>
            <h2>Heading 2, major sections of content</h2>
            <h3>Heading 3, subcategories/subtopics within a major section</h3>
            <h4>Heading 4, a subsection under a main section</h4>
            <h5>Heading 5, a subsection within a subsection</h5>
            <h6>Heading 6, etc</h6>
            </body>
            </html>

<header> Represents the introductory content of the page, typically containing a logo, the website's name, and navigation links. Using <h1>, <h2>, <h3>, etc., to organize content hierarchically is crucial for creating a well-structured and meaningful document outline. Each heading level represents a different level of importance or hierarchy in the content.

Navigation

            
            <!DOCTYPE html>
            <html lang="en">
            <head>
            <style>
            nav {
                background-color: #333;
                padding: 10px 0;
            }
            ul {
                list-style-type: none;
                padding: 0;
                margin: 0;
                text-align: center;
            }
            li {
                display: inline;
                margin-right: 20px;
            }
            a {
                color: #fff;
            }

            </style>
            </head>
            <body>
            <nav>
            <ul>
            <li><a href="link here">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Services</a></li>
            <li><a href="#">Contact</a></li>
            </ul>
            </nav>
            </body>
            </html>

<nav> Indicates a section of navigation links. By using the <nav> element, you provide semantic meaning to the navigation links, making it clear to both users and assistive technologies that these links are part of the website's navigation system. This enhances accessibility and helps users understand the purpose and function of the navigation section.

Main and Section

            
            <!DOCTYPE html>
            <html>
            <body>
            <main>
            <section>
            <h2>About Us</h2>
            <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit.</p>
            </section>
            <section>
            <h2>Contact Us</h2>
            <p>Contact us at contact@example.com or call us at +1234567890.</p>
            </section>
            </main>
            </body>
            </html>

<main> Contains the primary content of the webpage. While <section> Defines sections within the main content, each with a heading

In this example, the <main> element encloses two sections (<section> elements), each containing specific content related to the about and contact sections of the webpage. This indicates that these sections constitute the main content of the webpage.

Footer

            
            <!DOCTYPE html>
            <html lang="en">
            <head>
            <style>
            footer {
                background-color: #333;
                color: #fff;
                padding: 20px;
                text-align: center;
            }
            </style>
            </head>
            <body>
            <footer>
            &copy; 2024 My Website
            </footer>
            </body>
            </html>

<footer>typically appears at the bottom. It often contains information such as copyright notices, links to terms of service, privacy policies, contact information, and sometimes navigation links.
